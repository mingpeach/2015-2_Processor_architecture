The project is to make an optimal architecture parameter selection for superscalar processor in a fashion similar to the results shown in attached Text Chap. 3 (Sec.3.1-Sec.3.4).

  1. Run sim-outorder from SimpleScalar with two SPEC2006 programs. gcc and perl to decide on optimal processor architecture parameters, given I-Cache = 64KB and Dcache=16KB, and perfect branch prediction. “Optimal” here means a rational trade-off in cost and performance. Using ref data as input for the benchmark programs.

    a. The report should document your choice of the parameters qualitatively and quantitatively: Parameter values to choose are for Instruction issue width (and the number of function units), RUU (Register Update Unit) size, and D-cache associativity.

    Assume that the cycle time increases

    • 10% by doubling the issue width

    • 10% by doubling the RUU size

    • 2% by doubling the D-cache associativity

    • 2% by doubling the number of function units (including load/store units)

    b. Also, you need to make a cost model associated with hardware resource available so that you can make a trade-off in cost and performance.

  2. With the organization determined in 1, now you are asked to play with D-cache size and BTB size for branch prediction. We now use more practical bimodal predictor with its size choices of 512, 1K, 2K, and 4K entries. Assuming cycle time will increase 10% by doubling the size for D-cache (up to 128KB), try to decide the D-cache size along with the BTB size that provides the best performance. Choice of BTB size will not change cycle time but cost will change (you can apply the same cost model developed for 1 here).

  3. You can download the Simplescalar 3.0 source code from the course website. Change the host options in the Makefile accordingly before compiling the SimpleScalar executable(the simulator). Use the pre-compiled benchmark program binaries as the input benchmark programs for the SimpleScalar simulator. The input parameters for the benchmark programs are also available.

  4. Running all instructions in one program in SimpleScalar will typically cost more than 24 hours. So in this project, you are asked to skip first 1 billion instructions and just simulate the next 1 billion instructions for each benchmark programs. Please read the new features in the ANNOUNCE-3.0 in the simplesim-3.0 folder to figure out how to do it. (Hints: use –fastfwd and –max:instr options)
